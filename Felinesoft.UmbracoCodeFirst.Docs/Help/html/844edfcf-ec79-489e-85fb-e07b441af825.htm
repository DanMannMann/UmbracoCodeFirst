Working With Strongly-Typed Views

The Code-First package includes a special Razor view base class which provides access to strongly typed document models & a HtmlHelper instance configured to provide helper methods for the document type.

This topic contains the following sections:

  • Setting Up a View to Use Strongly-Typed Document Models
  • Implementing a Strongly-Typed View
Setting Up a View to Use Strongly-Typed Document Models

The UmbracoDocumentViewPageTdocument view base class is used with strongly-typed models. It inherits from UmbracoViewPage and provides everything you'd normally find in an Umbraco view, along with extra properties for the Document Model & the Document Helper.

To start using the view simply add the following line as the first line in your cshtml file, replacing any existing @model or @inherits declaration. T should be the Code-First type of the documents which use the view.

C#
Copy
@inherits Felinesoft.UmbracoCodeFirst.Views.UmbracoDocumentViewPage<T>
Implementing a Strongly-Typed View

In a strongly-typed view the @Document property can be used to access the strongly typed document. The familiar properties of an Umbraco view (@CurrentPage, @Model etc) continue to function as normal. The @DocumentHelper property provides access to a HtmlHelper<T>, where T is the document type. The properties, tabs and compositions of the document can be accessed as you'd expect. All of the non-collection-based built-in data types either implement IHtmlString or override ToString to give the correct output without accessing any properties or methods of the data type explicitly, as in the example below.

Copy
@inherits Felinesoft.UmbracoCodeFirst.Views.UmbracoDocumentViewPage<Felinesoft.UmbracoCodeFirst.Demo.DocTypes.TestPage>
@using Felinesoft.UmbracoCodeFirst.Demo.DocTypes;
@using Felinesoft.UmbracoCodeFirst.Linq;

@{
    Layout = "Layout.cshtml";

    //A strongly-typed model of the homepage node, obtained using the Code-First LINQ extension NearestAncestor<T>
    Home homepage = Document.NearestAncestor<Home>();
}
  <body>

    <!-- Page Wrapper -->
      <div id="page-wrapper">

        <!-- Header -->
          <header id="header">
            <h1><a href="/">@(homepage.SiteName)</a></h1>
            @{ Html.RenderPartial("umbNavigation", homepage.NavigationItems); }
          </header>

        <!-- Main -->
          <article id="main">
            <header>
              <h2>@Document.Content.Title</h2>
              <img src="@Document.Content.HeaderImage" />
            </header>
            <section class="wrapper style2">
              <div class="inner">
              <div class="byline">@Document.Content.Byline</div>
              <div class="article">@Document.Content.ArticleBody</div>
              </div>
            </section>
            <section class="wrapper style4">
              <div class="inner">
                <p>@Document.AddressLabel</p>
                <p>@DocumentHelper.DisplayFor(model => model.Address)</p>
              </div>
            </section>
            <section class="wrapper style2">
              <div class="inner">
              <a href="@Document.TypedContentPicker">Link to document @Document.TypedContentPicker.Name</a>
              </div>
            </section>
        <!-- Footer -->
        @{ Html.RenderPartial("umbFooter"); }
      </div>
