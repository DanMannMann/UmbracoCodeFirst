Development Roadmap

The development road map lists, in approximately priority order, the currently planned development work for Felinesoft Code-First for Umbraco

  • Change data type TypeConverters and interfaces to use a proper factory pattern. Modify same-class interface accordingly.

  • Decouple data types from document properties. Remove data type related properties from DocumentPropertyAttribute and modify core to get the values from the property type directly (move some code from DocumentPropertyAttribute.Initialise into core)

  • Refactor core to separate the various creation and update logic for the various entity types. Aim to increase extensibility and testability.

  • Add a [DataTypeInstance("Instance name")] property attribute to allow an instance of a data type to be created in-line on a document class, by applying the attribute to a property. [InstancePreValue], [InstancePreValueFile] and [InstancePreValueFactory] property attributes to be created allowing pre-values to be specified for an in-line data type instance in the same way as for a data type class.

  • Refactor support for media types, which has had no attention since before the project was forked from Umbraco Inception.

  • Add support for member types & user types

  • Improve support for template management

  • Add support for relationship types
