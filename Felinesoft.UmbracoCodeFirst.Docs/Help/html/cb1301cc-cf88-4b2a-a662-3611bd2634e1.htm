Data Type Interfaces

This topic contains the following sections.

  • Data Type Interfaces
  • Convenience Interfaces

The following interfaces can be used to specify the storage and underlying type of a custom data type. The underlying type is specified by the generic type parameter. The storage type depends on the interface used.

There are also a few convenience interfaces which specify data types using the standard mapping (i.e. where no property editor value type converter is used).

Data Type Interfaces

  • IUmbracoDateTimeDataTypeT

    Specifies that the data value should be stored as a DateTime2 type

  • IUmbracoIntegerDataTypeT

    Specifies that the data value should be stored as a 32-bit integer type

  • IUmbracoNtextDataTypeT

    Specifies that the data value should be stored as an Ntext type

  • IUmbracoNvarcharDataTypeT

    Specifies that the data value should be stored as an Nvarchar(MAX) type

Convenience Interfaces

  • IUmbracoDateDataType

    Specifies that the data value should be stored as a DateTime2 type and will be returned as a DateTime struct.

  • IUmbracoIntDataType

    Specifies that the data value should be stored as a 32-bit integer type and will be returned as a Int32 value.

  • IUmbracoStringDataType

    Specifies that the data value should be stored as an Ntext type and will be returned as a String value.
